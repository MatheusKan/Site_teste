Evolução do HTML do 1 ao 5

O HTML5 trouxe uma série de elementos novos e exclusivos que não estavam presentes nas versões anteriores do HTML. Esses elementos foram introduzidos para melhorar a semântica do código, facilitar a estruturação do conteúdo e oferecer suporte a recursos multimídia e interativos. Alguns dos elementos exclusivos do HTML5 incluem:

Semântica:

<"header">: Define o cabeçalho de uma seção ou página. <"nav">: Define uma seção de navegação. <"article">: Define um conteúdo independente que pode ser distribuído ou reutilizado. <"footer">: Define o rodapé de uma seção ou página. <"main">: Define o conteúdo principal da página. <"figure"> e <"figcaption">: São usados para incorporar imagens e figuras com legendas.


Formulários:

<"input type="email"">, <"input type="url"">, <"input type="tel"">: Tipos específicos de campos de entrada para email, URL e número de telefone. <"input type="range"">, <"input type="color"">, <"input type="date"">: Tipos de entrada para intervalo, cor e data. <"input type="search"">: Campo de busca.

Mídia:

<"audio"> e <"video">: Elementos para incorporar áudio e vídeo. <"source">: Utilizado dentro dos elementos <"audio"> e <"video"> para especificar diferentes fontes de mídia.



Armazenamento:

localStorage e sessionStorage: APIs de armazenamento local para salvar dados no navegador.

APIs JavaScript:

WebSockets: API para comunicação bidirecional em tempo real. Geolocalização: API para acessar informações de localização do dispositivo. Web Workers: API para executar scripts em segundo plano para melhorar a performance. IndexedDB: API de banco de dados orientado a objetos para armazenamento de dados estruturados. Web Storage: API para armazenamento de chave-valor no navegador.

Estruturação:

<"mark">: Destaca parte do texto. <"details"> e <"summary">: Usados para criar elementos de detalhamento que podem ser abertos ou fechados. <"time">: Representa datas e horários.
Links e navegação:

<"nav">: Elemento de navegação. <"a"> com atributo download: Permite que links baixem recursos ao invés de navegar para eles.

Recursos offline:

Service Workers: API para controlar ações e cache de recursos para permitir que páginas sejam acessadas offline. Esses são apenas alguns dos elementos exclusivos e recursos que o HTML5 trouxe para a web, tornando possível criar sites mais ricos, interativos e semânticos.
